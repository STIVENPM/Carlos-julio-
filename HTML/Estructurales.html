¿cual es la estructuras en HTML?

La estructura de un documento HTML se basa en una jerarquía de etiquetas que organiza el contenido en dos secciones fundamentales contenidas dentro de la etiqueta raíz head.La primera es el head, que funciona como el cerebro del documento almacenando metadatos, configuraciones de caracteres, el título de la pestaña y enlaces a recursos externos que no son visibles para el usuario. La segunda es el body, que constituye el cuerpo visible de la página y donde se declaran todos los elementos estructurales como encabezados, párrafos, imágenes e hipervínculos. Esta arquitectura, encabezada siempre por la declaración !DOCTYPE html, garantiza que el navegador interprete correctamente la semántica y el propósito de cada componente de la interfaz.

Tipos de estructuras

Sirven para organizar la estructura de la página web estas se diveden en dos partes :

Estructuras de configuración (El contenedor)

  • html:

    • Es la etiqueta raíz o "padre" de todo el documento. Representa el inicio y el fin de la página web. Todo lo que escribas debe estar dentro de esta etiqueta.

  • head:

    • Es la sección de configuración. Aquí se guarda la información "invisible" para el usuario, como el título que aparece en la pestaña del navegador, los enlaces a archivos CSS o las palabras clave para Google.

Estructuras de contenido y diseño (Lo visible)

  • body:

    • Es el cuerpo de la página. Todo lo que pongas aquí (textos, imágenes, botones) es lo que el usuario final verá y con lo que podrá interactuar en su pantalla.

  • nav:

    • Se utiliza específicamente para bloques que contienen enlaces de navegación (menús). Ayuda a los lectores de pantalla para personas con discapacidad visual a identificar dónde está el menú principal.

  • section:

    • Define una sección genérica dentro del documento. Se usa para agrupar contenido relacionado (por ejemplo: una sección de "Servicios", otra de "Testimonios" o una de "Contacto").

  • footer:

    • Es el pie de página. Se ubica al final del documento y suele contener información de copyright, enlaces a redes sociales, políticas de privacidad o datos de contacto.
